# Options

These are the additional options that can be added to the TypeDoc configuration.

## frontmatterCommentTags

<Callout>Specify which comment block tags should be added to frontmatter.</Callout>

> Accepts an Array.


```yaml filename="YAML"
---
layout: docs
sidebar: true
---
```


```json filename="typedoc.json"
{
  "frontmatterCommentTags": {
    "layout": "docs",
    "sidebar": true
  }
}

```

___

## frontmatterGlobals

<Callout>Specify static variables to be added to all frontmatter blocks.</Callout>

> Accepts a key/value object.

Frontmatter variables can be added by extracting comments from block (@) tags.

Please note tags must be added to the comment blocks of the symbol exported to a page.

 ```ansi filename="Block Tags (someModule.ts)"
 /**
 * @author Joe Bloggs
 *
 * @description A description that will be added to frontmatter.
 */
 ```

```yaml filename="YAML (someModule.md)"
---
author: Joe Bloggs
description: A description that will be added to frontmatter.
---
```


```json filename="typedoc.json"
{
  "frontmatterGlobals": [
    "author",
    "description"
  ]
}

```

___

## frontmatterNamingConvention

<Callout>The naming convention that variables should be output as. </Callout>

> Accepts either `"camelCase"` or `"snakeCase"`. Defaults to `"camelCase"`.


```json filename="typedoc.json"
{
  "frontmatterNamingConvention": "camelCase"
}

```

___

## preserveFrontmatterCommentTags

<Callout>Preserve tags defined in frontmatter block tags in output.</Callout>

> Accepts a boolean value. Defaults to `false`.

This option can configure the output style of frontmatter variables.

Note that block tags should be written in camelCase.


```json filename="typedoc.json"
{
  "preserveFrontmatterCommentTags": false
}

```
